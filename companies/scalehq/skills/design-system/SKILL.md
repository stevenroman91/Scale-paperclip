# Scale Design System

Skill de reference du design system de ScaleHQ — Tailwind CSS + Radix UI.

## Capacites

- **Tokens Tailwind** : reference des tokens de design definis dans `tailwind.config.ts` — palette de couleurs (primary, secondary, accent, neutral, success, warning, error), typographie (font families, sizes, weights, line-heights), espacements, border-radius, shadows, breakpoints.
- **Composants Radix UI** : catalogue des composants Radix UI utilises dans ScaleHQ avec leurs variantes, leurs props, et leurs patterns d'utilisation — Dialog, Popover, Select, Tabs, Tooltip, DropdownMenu, AlertDialog, Toast, Checkbox, RadioGroup, Switch, Slider, etc.
- **Patterns de layout** : patterns de mise en page recurrents — sidebar layout, dashboard grid, data table, form layout, card layout, empty state, loading state, error state. Chaque pattern avec son code de reference.
- **Guidelines d'accessibilite** : regles d'accessibilite a respecter — contrastes de couleur (WCAG 2.1 AA), navigation clavier, labels ARIA, focus visible, tailles de cible tactile, textes alternatifs, roles semantiques.
- **Conventions de nommage** : conventions pour les noms de classes, les noms de composants, et la structure des fichiers — `cn()` pour composer les classes Tailwind, PascalCase pour les composants, kebab-case pour les fichiers.
- **Icones** : reference du set d'icones lucide-react utilise dans l'application, avec les conventions d'utilisation (taille, couleur, espacement).
